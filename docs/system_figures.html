Figure 1. FLUX v0.1 System Architecture. The system follows a layered architecture with a Next.js presentation layer communicating via REST/SSE with a FastAPI application layer. The service layer provides RAG-based retrieval, LLM routing with per-step model assignment, and knowledge graph operations. Data is persisted across PostgreSQL (with pgvector for 768-dimensional embeddings), Neo4j (ontology graph), and Redis (caching). AP = Account Planner, BS = Brand Strategist, CD = Creative Director, PD = Presentation Designer.

Figure 2. Service Flow Diagram. The upper section shows the end-to-end user journey from brief input through director selection and pipeline execution to slide output. The middle section details the SSE event stream protocol: each pipeline step emits a sequence of typed events enabling real-time team discussion visualization. The lower-left shows the JWT-based authentication flow with automatic guest registration. The lower-right shows available export paths.

Figure 3. 3-Layer Ontology and RAG Hybrid Search Pipeline. The knowledge graph comprises four node types (Method, Case, Director, Industry) connected by five edge types. Layer 1 (Methods) contains 61 strategy frameworks with 768-dimensional vector embeddings. Layer 2 (Cases) stores 21 campaign case studies linked to methods via APPLIED_IN edges. Layer 3 (Directors) represents 6 creative persona archetypes with weighted preference dimensions. The RAG pipeline performs hybrid retrieval: vector search (pgvector cosine similarity) → graph enrichment (Neo4j co-occurrence traversal) → director preference boosting → re-ranked composite scoring.

Figure 4. 8-Step Pipeline with Multi-Agent Team Discussion. The pipeline executes sequentially across three phases: Analysis (Account Planner lead), Strategy (Brand Strategist lead), and Creative (Creative Director lead). Each step employs a structured team discussion pattern where the selected director persona frames the discussion, the lead agent provides primary analysis with optional RAG retrieval, support agents contribute reactions, and the director synthesizes the final output. Method retrieval via RAG occurs at steps S4 and S6; case retrieval timing varies by director archetype per the case_timing policy. The Presentation Designer generates the final slide deck from all 8 step outputs.
